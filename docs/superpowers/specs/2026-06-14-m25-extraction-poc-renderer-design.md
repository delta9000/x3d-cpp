# M2.5 Extraction API + PoC OpenGL Renderer — Design

**Date:** 2026-06-14  
**Status:** design complete (workflow `wf_e54b997b-a5b`: 5 research dims → synthesis → 3 adversarial critiques [0/3 solid first round, 10 blocker/major folded in] → revise).  
**Provenance:** 10 agents, ~607k tokens. This supersedes nothing; it opens the M2.5 milestone per the resequenced roadmap.

## Design summary

M2.5 is a header-only, golden-untouched, node-as-truth PULL extraction seam (new runtime/extract/ namespace x3d::runtime::extract) layered as a read-only adapter on top of X3DExecutionContext's existing public pull surface, plus a SEPARATE out-of-SDK OpenGL PoC consumer at examples/poc_renderer/ (opt-in, firewalled behind X3D_CPP_BUILD_POC=OFF so the default build/golden/ctest path is untouched). The unifying decision that resolves the scheduled BACKLOG M2C-1 fork is: GEOMETRY and MATERIAL are node-keyed (legitimately shared by USE, frame-invariant, upload-once / instance-N), while WORLD TRANSFORM and therefore RenderItem IDENTITY are per-PATH. One DEF'd Shape USE'd under two Transforms yields TWO RenderItems sharing one GeomId+MaterialDesc-by-value but with distinct PathKeys and distinct worldTransforms. The extractor NEVER reads TransformSystem.world_ (the verified M2C-1 first-path overwrite at TransformSystem.hpp:92) for RenderItem matrices: it accumulates worldM fresh down each placement exactly as PickSystem::worldOfRec already proves works, and crucially it CACHES each interior path-node's accumulated entry-matrix so incremental updates are O(path)/O(subtree), not O(scene). Five blocker/major corrections from the adversarial review are folded in: (1) the reverse index is THREE maps (transformDeps / geomDeps / materialDeps) so a Material/Appearance node change — which classifyDirty marks only on the owning node, below the Shape — actually resolves to its RenderItems; (2) DirtyChildren re-walks resume from the cached per-path entry matrix, never from the buffered per-node table; (3) the seam contract is ONE tick : ONE delta() (asserted), since X3DExecutionContext.tick() clears the dirty set at line 103 — multiple ticks between deltas would silently drop changes; (4) updatedTransform RE-ACCUMULATES worldM along the stored PathKey (O(path)), never reads ctx.worldTransform() which is first-path-only; (5) updatedGeometry is gated on a CONTENT field-name change, never on the coarse DirtyBounds bit (which BoundsSystem.recomputeUp sets during pure ancestor-scale animation, polluting it). Spec-fidelity is corrected too: DirectionalLight default global=false (verified) so the PoC's no-explicit-lights fallback is the bound NavigationInfo headlight (default true), not a scene-wide promotion; Switch uses whichChoice (default -1 = draw nothing) and LOD uses children/range with a documented level-0 static selection — the visibility DFS special-cases these by nodeTypeName BEFORE the generic child loop rather than reusing PickSystem's blind forEachChild. The task plan is re-ordered into a TRUE vertical slice: a minimal extractor over only the three exact types PickSystem already handles (IFS/IndexedTriangleSet/TriangleSet) with an Unlit-white material stub and no lights gets an unlit triangle on Wayland after T0+T1+T7a+T9; the full mesh/material/light/visibility stack are incremental widenings behind an already-rendering pipeline. PoC scope is honest: unlit-flat first light, then lit; textures, HAnim, OrthoViewpoint, GeoViewpoint, NURBS, Text, Extrusion explicitly deferred.

## Architecture

THREE LAYERS, all header-only, ZERO edits to generated_cpp_bindings and zero edits to X3DExecutionContext's existing members (the extractor is a free const& consumer over the existing public surface).

=== LAYER A — runtime/extract/ (the M2.5 seam, namespace x3d::runtime::extract) ===

A1. RenderItem.hpp (pure POD descriptors; depends only on X3Dtypes + Mat4 + Aabb + std):
  - PathKey = std::vector<const X3DNode*> (root..leaf). Identity is the FULL vector with full-vector equality; the hash is ONLY a hashmap bucket key, NEVER identity (closes the hash-collision-aliasing flaw). RenderItemId = a dense uint32 handle into the extractor's path-interning table (stable across frames while topology stable), NOT a raw hash.
  - GeomId { const X3DNode* node; uint32 contentVersion; } — content-keyed; equal GeomId across placements => upload-once / instance.
  - MeshData { std::vector<SFVec3f> positions; std::vector<uint32_t> indices; std::vector<SFVec3f> normals; std::vector<SFVec2f> texcoords; std::vector<SFColorRGBA> colors; bool ccw=true; bool solid=true; bool hasNormals; bool hasColors; }. positions/normals are LOCAL-frame (never world-baked). indices ALWAYS populated (trivial 0..N-1 when expanded). LOCAL texcoord origin is bottom-left = GL convention, so NO V-flip (documented in header so an implementer doesn't reflexively flip).
  - MaterialDesc (tagged POD, shared BY VALUE across RenderItems; NOT part of identity): enum MaterialModel{Phong,Physical,Unlit}; SFColorRGBA baseColor (diffuse or baseColor composed with alpha=1-transparency); SFColor emissive,specular; float metallic,roughness,shininess,ambientIntensity,occlusionStrength; enum AlphaMode{Opaque,Mask,Blend}; float alphaCutoff; bool unlit; std::vector<TextureRef> textures; SFColorRGBA toRGBA() — header DOCUMENTS alpha = 1.0f - transparency. Null material under a present Appearance => Phong diffuse=0.8 grey (spec default Material); null Appearance entirely => Unlit white emissive (absolute always-draws debug fallback, flagged non-spec in the header).
  - TextureRef { enum Slot{BaseColor,Normal,Emissive,Occlusion,MetallicRoughness,Specular,Diffuse}; enum Source{Url,Inline,Movie}; MFString url; bool repeatS,repeatT; SamplerParams sampler; SFImage inlinePixels; int channel; } — url verbatim (asset resolution is OUTSIDE the SDK); PixelTexture => Inline pixels; MovieTexture => Movie tag.
  - LightDesc { enum Type{Directional,Point,Spot}; SFColor color; float intensity,ambientIntensity; SFVec3f worldDirection,worldLocation; SFVec3f attenuation; float radius,beamWidth,cutOffAngle; bool global; const X3DNode* scopeRoot; } — scopeRoot is the grouping node the light was collected under, carried so a non-global light can be scoped to descendants of its PathKey ancestor.
  - CameraDesc { Mat4 viewMatrix; float fieldOfView; float nearPlane,farPlane; bool ortho; bool cameraChanged; } — fieldOfView is X3D's MIN-dimension FOV (documented; consumer maps to vertical/horizontal by aspect). viewMatrix is first-path-resolved (documented caveat).
  - BackgroundDesc { std::vector<SFColor> skyColor; std::vector<float> skyAngle; std::vector<SFColor> groundColor; std::vector<float> groundAngle; bool backgroundChanged; }.
  - RenderDelta { std::vector<RenderItemId> added, removed, updatedTransform, updatedGeometry, updatedMaterial; bool cameraChanged, backgroundChanged, lightsChanged; }. This is THE single authoritative change channel; RenderItem carries NO changeBits (dropped — no parallel encoding). fullSnapshot() returns a RenderDelta with every item in `added` (so frame-0 and frame-N share ONE upload path).

A2. MeshBuilder.hpp (geometry -> MeshData; promoted+extended from PickSystem::extractTriangles; depends only on X3DNode + X3Dtypes + geombounds::getField/getNode/hasField — NO Bounds/Transform dep, no cycle):
  - buildLocalMesh(const X3DNode* geom, const MeshBuildOptions&) -> MeshData. Single addTriangle funnel so pick + render share ONE traversal + ONE triangulation.
  - Tier-1 (T1, the three EXACT types PickSystem proves today): IndexedFaceSet (coordIndex fan), IndexedTriangleSet (index triples), TriangleSet (implicit triples). ok()-bounds-guard on every index (corpus only ~93-99% clean).
  - Tier-2 (T2): IndexedTriangleFanSet/StripSet/QuadSet, TriangleFanSet(fanCount)/StripSet(stripCount)/QuadSet, ElevationGrid(height/xDimension/zDimension/xSpacing/zSpacing). Per-triangle strip winding flip (explicit, tested).
  - Tier-3 (T4): Box/Sphere/Cone/Cylinder parametric tessellation (MeshBuildOptions density: sphere 16x16, cone/cyl 24 radial), CCW outward normals so they compose with the same cull/two-sided rule as meshes.
  - Attributes (T3): resolve authored Normal(vector)/Color(color: MFColor)/ColorRGBA(color: MFColorRGBA)/TextureCoordinate(point) honoring normalPerVertex/colorPerVertex + normalIndex/colorIndex/texCoordIndex. SFColor promoted to SFColorRGBA(alpha=1). Per-vertex Color OVERRIDES Material diffuse per X3D lighting (documented at the seam; MeshData.hasColors signals it). FLAT per-face normals generated when no Normal node. ccw/solid carried via reflection getField<SFBool>(geom,"solid"/"ccw",true) (inherited from X3DComposedGeometryNode, verified).
  - Buffer-expansion policy: keep source indices for natively-indexed types; EXPAND to non-indexed (trivial 0..N-1) when flat/per-face normals or *PerVertex=false break vertex sharing. Builder owns the decision off the actual *PerVertex flags.
  - Deferred (return empty MeshData, log-once): Text, Extrusion, NURBS, GeoElevationGrid, Point/LineSet.

A3. MaterialSystem.hpp (stateless free functions):
  - materialOf(const X3DNode* appearance) -> MaterialDesc, dispatching on material->nodeTypeName() but reading reflection-generic getField<SFColor>/<SFFloat> by SPEC FIELD NAME (so a future TwoSidedMaterial reuses the Phong path with zero edits — no dynamic_cast). Material => diffuseColor/transparency/emissiveColor/specularColor/shininess/ambientIntensity; PhysicalMaterial => baseColor/metallic/roughness/emissiveColor/transparency. alphaMode/alphaCutoff read off the Appearance. Null material + present Appearance => Phong 0.8 grey; null Appearance => Unlit white.
  - texturesOf(const X3DNode* appearance) -> std::vector<TextureRef>: material-slot textures (baseTexture/emissiveTexture/...) take precedence over legacy Appearance.texture; url verbatim ordered fallback; sampler from textureProperties; PixelTexture => Inline; MovieTexture => Movie; MultiTexture => ascending channels.

A4. LightSystem.hpp (copies PickSystem roots_/forEachChild/worldOf accumulation idiom):
  - collect(const Scene&) -> std::vector<LightDesc>: walk roots accumulating worldM; for every node whose nodeTypeName is in {DirectionalLight,PointLight,SpotLight} (or generically hasField on/color/intensity) with on==true, emit LightDesc with worldDirection=worldM.transformDirection(direction).normalized and worldLocation=worldM.transformPoint(location), reading color/intensity/ambientIntensity/attenuation/radius/beamWidth/cutOffAngle/global generically. scopeRoot = the nearest enclosing grouping node. World-resolved AT COLLECTION => sidesteps M2C-1 entirely. The authored `global` (DirectionalLight default false, verified) is CARRIED, not silently promoted.

A5. SceneExtractor.hpp (the pull+delta engine; holds const X3DExecutionContext&; ALL caches are side tables — zero node state):
  - PathIndex: interning table mapping PathKey (full-vector-equality) -> RenderItemId; plus per-RenderItem stored data { PathKey path; Mat4 worldTransform; GeomId geometry; MaterialDesc material; }.
  - INTERIOR-NODE ENTRY-MATRIX CACHE: during every full/subtree walk, store the accumulated worldM at each interior path-node (keyed by that node's path-prefix). This is what makes a DirtyChildren subtree re-walk resume from the correct entry matrix in O(subtree) without ever reading TransformSystem.world_ (closes the entry-matrix blocker).
  - THREE REVERSE INDICES (closes the dead-material-channel blocker): transformDeps (Transform path-ancestor node -> RenderItemIds whose path contains it), geomDeps (geometry node -> RenderItemIds), materialDeps (every appearance-subtree node: appearance, material, each texture/textureProperties node reachable from the Shape -> RenderItemIds). Built during the DFS when each Shape RenderItem is emitted.
  - VISIBILITY-AWARE DFS (the real gap beyond Pick — does NOT reuse forEachChild blindly): special-case BY nodeTypeName BEFORE the generic child loop. Switch: read whichChoice (SFInt32, default -1); if <0 or >=children.size() emit nothing, else recurse ONLY children[whichChoice]. LOD: static PoC selects children[0] (highest detail), documented (range selection needs an eye position not threaded into the DFS). Billboard/Collision/StaticGroup/Group/Transform: pass-through groups (Billboard treated as identity orientation — documented wrong-once-camera-moves). worldM mutated ONLY by Transform (isTransform == nodeTypeName=="Transform"); HAnimJoint/CADPart frames dropped (M2C-4, HAnim out of PoC scope). The generic child recursion is gated to grouping/children MFNode + geometry/appearance SFNode slots, NEVER metadata. RenderItem emission anchored strictly on geombounds::hasField(n,"geometry") at a Shape (exactly PickSystem::pickNode), not on DFS leafness.
  - fullSnapshot() -> RenderDelta: one full walk; emits one RenderItem per (geometry-bearing leaf, path); populates PathIndex + entry-matrix cache + three reverse indices + GeomId mesh cache (delegating to MeshBuilder, versioned) + MaterialDesc cache (delegating to MaterialSystem). Returns ALL items as `added`.
  - delta() -> RenderDelta (reads ctx.dirtyTracker().changedNodes()+flags()): for each changed node n —
      * DirtyLocalTransform/DirtyWorldTransform on a Transform => for each RenderItemId in transformDeps[n], RE-ACCUMULATE that item's worldTransform by re-multiplying TransformSystem::localMatrix along its stored PathKey (O(path)); add to updatedTransform. NEVER reads ctx.worldTransform() (first-path-only). Transform-only ticks do NOT re-walk.
      * DirtyField on a geometry node AND the changed field is a CONTENT field (coord/coordIndex/point/height/radius/size/index/...) => bump GeomId.contentVersion, re-extract mesh, geomDeps[n] => updatedGeometry. A bare DirtyBounds bit (AABB-only signal from BoundsSystem ancestor recompute) does NOT trigger re-extract (closes the pollution flaw). Field-name classification mirrors classifyDirty's kBounds-content subset.
      * DirtyField on an appearance-subtree node => materialDeps[n] => re-read MaterialDesc => updatedMaterial.
      * DirtyChildren on a grouping node G => look up every PathIndex prefix ending at G, resume the subtree walk from each with its CACHED entry matrix => added/removed by diffing the prior vs new path set for that subtree.
  - camera() -> CameraDesc: viewMatrix from ctx.viewMatrix(); fieldOfView from getField<SFFloat>(*ctx.boundViewpoint(),"fieldOfView",0.7854f); if nodeTypeName()=="OrthoViewpoint" set ortho=true and read its MFFloat fieldOfView (l/b/r/t) — OrthoViewpoint projection params are a different shape and are surfaced but PoC-out-of-scope (documented). near/far are NOT computed here (no single root); the PoC fits them from sceneWorldBounds() (below).
  - sceneWorldBounds() -> Aabb: union over every RenderItem of (its local mesh AABB transformed by item.worldTransform). This is per-path-correct and is what the PoC feeds near/far, NOT ctx.worldBounds(root) which is first-path and undercounts the second USE placement (closes that major).
  - lights() -> std::vector<LightDesc> (recomputed full each tick for the PoC — few; cheap; lightsChanged always true at PoC stage). background() -> BackgroundDesc (full each tick; backgroundChanged surfaced for a caching consumer).
  - SEAM CONTRACT (asserted invariant): exactly ONE delta() per tick(); batching ticks between deltas drops changes because tick() clears dirty_ at line 103. The extractor records the tick generation count and asserts delta() is called once per advance. THREADING CONTRACT: single-threaded producer+consumer; the extractor's mutable caches are not safe to share across threads (stated as a seam invariant).

=== LAYER B — X3DExecutionContext additions ===
NONE required. The extractor consumes the existing public surface (dirtyTracker, worldTransform [used only for documented first-path queries, never for RenderItem matrices], localBounds, boundViewpoint/Background/NavigationInfo, viewMatrix, pick). X3DExecutionContext stays byte-untouched to preserve golden.

=== LAYER C — examples/poc_renderer/ (out-of-SDK consumer, target x3d_poc_renderer) ===
  - examples/poc_renderer/CMakeLists.txt gated behind option(X3D_CPP_BUILD_POC OFF); root CMake does add_subdirectory ONLY when ON, so default `mise run build`/ctest/golden are untouched and it never joins the OOM-prone all-headers TU. Links INTERFACE alias x3d_cpp::x3d_cpp + glfw (FetchContent pinned >=3.4, GLFW_BUILD_WAYLAND=ON, GLFW_BUILD_X11=ON as an XWayland safety net) + PRE-GENERATED glad2 sources committed at examples/poc_renderer/third_party/glad/ (glad.c, glad/gl.h, KHR/khrplatform.h — real files, not a generator FetchContent). GL core profile 3.3 (GLSL 330).
  - main.cpp: parseFile(argv[1]) -> getScene -> ctx.buildSceneGraph(s); BridgeResult br = ctx.buildFrom(s) — CHECK br for unresolved EXTERNPROTO/bad ROUTE and bail/log on failure (needs X3DSceneBridge.hpp). Construct SceneExtractor(ctx). GLFW init with explicit hints: CONTEXT_VERSION 3.3, OPENGL_CORE_PROFILE, OPENGL_FORWARD_COMPAT, DEPTH_BITS 24. glad init. GL_KHR_debug callback wired + a forced fullscreen-clear sanity step (so a black screen is diagnosable). Per frame: ctx.tick(now); frame 0 => fullSnapshot() else delta() (exactly one per tick); upload MeshData per GeomId ONCE into a GpuMesh cache keyed by GeomId (glBufferData orphan on updatedGeometry); per RenderItem set model=item.worldTransform (column-major Mat4 uploaded directly via m.m.data()), view=camera().viewMatrix, projection synthesized PoC-side from CameraDesc.fieldOfView (mapped to the SHORTER window dimension per X3D min-FOV rule) + aspect, near/far fit from extractor.sceneWorldBounds(); GC GpuMeshes whose GeomId left the set.
  - Culling: GL_CULL_FACE DISABLED for M1/M2 (draw double-sided until winding proven); enabled at M3 honoring MeshData.ccw (ccw=true => glFrontFace(GL_CCW)) + solid=true => cull back. Lit shader uses two-sided lighting (N = gl_FrontFacing ? N : -N) so flat normals shade correctly regardless of winding.
  - No-lights fallback: if lights() is empty AND boundNavigationInfo headlight==true (default), the PoC uses a camera-space headlight (the spec-correct X3D no-lights case) instead of promoting a global=false DirectionalLight.
  - Shaders: flat.vert/frag (M1/M2 single color from MaterialDesc.toRGBA or per-vertex Color when hasColors), lit.vert/frag (M3 Lambert+ambient from normals + DirectionalLight/headlight LightDesc honoring diffuse/emissive/specular/transparency).
  - Coordinate system: X3D right-handed Y-up passes straight through to GL (no axis remap). Texture origin bottom-left = GL (no V-flip).

INVARIANTS HELD: header-only (every SDK file is .hpp; only the PoC has .cpp, outside the SDK); golden-untouched (no edits to generated_cpp_bindings; PickSystem re-seat in T1 is pinned to EXACTLY its current three exact types with the rayAabb fallback retained for all others => pick goldens bit-identical; T4 primitive mesh-picks are an OPT-IN follow-up with their OWN regenerated goldens, NOT folded under "behavior-identical"); node-as-truth (extractor stores nothing on nodes); headless SDK / renderer-as-consumer (the renderer links only the INTERFACE lib across the seam).

## Decisions resolved (sensible defaults, not user-gated)

- **RenderItem identity: per-node vs per-PATH (BACKLOG M2C-1, the scheduled pre-M2.5 spike)** — PER-PATH for transform/identity, PER-NODE for geometry+material. RenderItemId is a dense handle into a path-interning table with FULL-PathKey equality (the hash is only a hashmap bucket, never identity — closes the collision-aliasing flaw). worldTransform is accumulated in a fresh visibility-aware DFS and CACHED per interior path-node, NEVER read from TransformSystem.world_ (verified first-path overwrite at TransformSystem.hpp:92). GeomId/MaterialDesc keyed by node* (USE-shared content identical => upload once, instance N). Verified low-risk: the per-path accumulation already exists in PickSystem::worldOfRec.
- **Pull vs push at the seam (the one decision spec deferred to M2.5)** — PULL, confirmed. tick(now) then fullSnapshot() (frame 0) / delta() (thereafter). CONTRACT: exactly ONE delta() per tick(), asserted — because X3DExecutionContext.tick() clears dirty_ at line 103 (verified), batching ticks between deltas would silently drop changes. Single-threaded producer+consumer is a seam invariant.
- **Material/Appearance change reverse-resolution (the dead-channel blocker)** — The reverse index is THREE maps, not one PathKeys map: transformDeps, geomDeps, materialDeps. classifyDirty (verified) marks a Material/Appearance field change ONLY on the owning node, which sits BELOW the Shape and is neither a path ancestor nor the renderable leaf. materialDeps records every appearance-subtree node (appearance, material, textures, textureProperties) reachable from each Shape -> its RenderItemIds, so updatedMaterial actually fires. The three maps have distinct invalidation semantics.
- **DirtyChildren subtree re-walk entry transform (the entry-matrix blocker)** — During every full/subtree walk the extractor CACHES each interior path-node's accumulated entry worldM (keyed by path-prefix). On DirtyChildren at grouping node G, resume the subtree walk from each PathIndex prefix ending at G using its cached entry matrix — O(subtree), never reading TransformSystem.world_. This was unaddressed in the original design; folded in.
- **updatedTransform recompute (the incremental-staleness hole)** — On a transform-only tick, for each RenderItem flagged updatedTransform, RE-ACCUMULATE its worldTransform by re-multiplying TransformSystem::localMatrix along the stored PathKey (O(path length)). NEVER read ctx.worldTransform() (first-path-only, verified). This keeps the incremental promise AND is per-path-correct for USE-shared/animated subtrees. No subtree re-walk for transform-only ticks.
- **updatedGeometry trigger (the DirtyBounds-pollution major)** — Bump GeomId.contentVersion ONLY on a DirtyField whose field name is in the geometry-CONTENT subset (coord/coordIndex/point/height/radius/size/index/...). NEVER on the coarse DirtyBounds bit — verified that BoundsSystem.recomputeUp marks DirtyBounds on participating nodes during pure ancestor-scale animation and classifyDirty adds DirtyBounds for the Transform 'scale' field, which would needlessly re-extract+re-upload unchanged meshes every frame. DirtyBounds (AABB-only) is split from content-DirtyField (mesh) at the seam.
- **Camera/projection params source** — viewMatrix from ctx.viewMatrix() (documented first-path caveat — the bound Viewpoint is rarely USE'd under two frames). fieldOfView read by the extractor directly from the bound Viewpoint node via getField<SFFloat>(*boundViewpoint(),'fieldOfView',0.7854f) — verified there is NO fov accessor on X3DExecutionContext. OrthoViewpoint (verified fieldOfView is MFFloat {-1,-1,1,1} l/b/r/t) sets ortho=true, surfaced but PoC-out-of-scope. fieldOfView is the X3D MIN-dimension FOV; the PoC maps it to the shorter window dimension (documented), not naively to vertical FOV.
- **Near/far + scene bounds for projection fit** — The extractor exposes sceneWorldBounds() = union of every RenderItem's local mesh AABB transformed by its per-path worldTransform. The PoC fits near/far from THAT, NOT ctx.worldBounds(root) (first-path, verified to undercount the second USE placement — the exact case the PoC demonstrates). Default near=0.1/far=large when bounds empty.
- **Geometry identity & dedup** — GeomId = geometry node* + contentVersion. Mesh uploaded once, instanced across placements. Per-node here is CORRECT (frame-invariant, legitimately USE-shared). MaterialDesc is computed per (appearance node) and shared BY VALUE across RenderItems but is NOT part of identity (resolves the per-path-vs-node-keyed conflict the critic flagged).
- **Where the shared mesh builder lives + golden-safety of the PickSystem re-seat** — Promote extractTriangles into runtime/extract/MeshBuilder.hpp (depends only on X3DNode + X3Dtypes + geombounds — no cycle). PickSystem::narrowPhase re-seated to build the local mesh once for EXACTLY its current three types (IFS/IndexedTriangleSet/TriangleSet), keeping the Sphere analytic fast-path AND the rayAabb fallback for everything else => pick goldens BIT-IDENTICAL. The T4 primitive mesh-pick upgrade is a SEPARATE opt-in follow-up with regenerated goldens — NOT under the behavior-identical banner (corrects the original's contradiction).
- **Default normal generation** — FLAT per-face normals when no Normal node; honor authored Normal; defer creaseAngle smoothing. Lit shader uses two-sided lighting so flat normals shade correctly regardless of winding.
- **Indexed vs expanded buffer layout** — Keep source indices for natively-indexed types; expand to non-indexed (trivial 0..N-1) when flat/per-face normals or *PerVertex=false break sharing. Builder owns the decision off actual *PerVertex flags. MeshData.indices always populated.
- **Material model surface** — ONE tagged MaterialDesc read reflection-generic by spec field name keyed on nodeTypeName() (no dynamic_cast; future TwoSidedMaterial reuses Phong path with zero edits). toRGBA() DOCUMENTS alpha = 1.0 - transparency (verified Material default transparency=0 => alpha=1; naive alpha=transparency would make defaults invisible). Null material+present Appearance => Phong 0.8 grey; null Appearance => Unlit white (flagged non-spec debug fallback). Per-vertex Color OVERRIDES Material diffuse per X3D lighting.
- **Textures at the seam** — Surface MFString url verbatim + sampler, NOT bytes (asset resolver outside SDK). PixelTexture=>Inline pixels; MovieTexture=>Movie; MultiTexture=>ascending channels. Material-slot texture wins over legacy Appearance.texture (deterministic, matches 4.0 intent, documented so 3.x still textures). No V-flip (X3D origin bottom-left = GL).
- **Light world-resolution + scoping (M2C-1's shadow + the spec-fidelity major)** — Each light's world transform resolved by fresh root-walk accumulation (PickSystem idiom), correct per-placement regardless of M2C-1. CRITICAL: verified DirectionalLight default global=false (PointLight global=true). LightDesc carries the authored global AND a scopeRoot; the PoC does NOT silently promote a global=false light to scene-wide. The PoC no-explicit-lights fallback is the bound NavigationInfo headlight (default true, verified) as a camera-space directional — the spec-correct X3D no-lights case.
- **Changed-set granularity (DirtyFlag -> RenderDelta)** — RenderDelta is the single authoritative change channel; RenderItem carries NO changeBits (dropped the parallel encoding). fullSnapshot() returns all items as 'added' (one upload path for frame 0 and N). DirtyLocalTransform=>updatedTransform (re-accumulate path); content-DirtyField=>updatedGeometry; appearance-subtree DirtyField=>updatedMaterial; DirtyChildren=>subtree re-walk=>added/removed. camera/background/lights recomputed full with cameraChanged/backgroundChanged/lightsChanged bits surfaced for a caching consumer.
- **Visibility-aware traversal (the real gap beyond Pick + the spec field-name major)** — The DFS special-cases by nodeTypeName BEFORE the generic child loop; it does NOT reuse PickSystem's blind forEachChild for Switch/LOD. Verified: Switch field is whichChoice (NOT choice), default -1 => draw nothing; <0 or out-of-range emits nothing, else recurse only children[whichChoice]. LOD fields are children/range (NOT level); static PoC selects children[0] (documented, range needs an eye position not threaded into the DFS). Billboard/Collision/StaticGroup pass-through; only Transform mutates worldM. Generic recursion gated to grouping/geometry/appearance node slots, never metadata.
- **PoC windowing / GL / loader / target placement** — GLFW>=3.4 FetchContent (Wayland on, X11 ON as XWayland safety net), OpenGL core 3.3 (GLSL 330; RTX 4080 trivial), PRE-GENERATED glad2 sources committed as real files (not a generator FetchContent). New examples/poc_renderer/ target linking only INTERFACE alias x3d_cpp::x3d_cpp, gated option(X3D_CPP_BUILD_POC OFF) so default build/golden/ctest untouched. Explicit GLFW hints (core profile, forward-compat, 24-bit depth) + GL_KHR_debug callback + forced-clear sanity step for black-screen diagnosis.
- **Projection matrix ownership** — PoC computes its own perspective from CameraDesc.fieldOfView + window aspect (fieldOfView mapped to the SHORTER dimension per X3D min-FOV rule); near/far from extractor.sceneWorldBounds(). The SDK has no projection helper and legitimately shouldn't (aspect is a window property).
- **Vertical-slice ordering (the 'first-light is the LAST thing' major)** — Re-ordered: a minimal extractor (T7a) over ONLY T1's three exact types + Unlit-white material stub + NO LightSystem + NO visibility logic gets an unlit triangle on screen after T0+T1+T7a+T9, against a hand-written triangle .x3d. T2/T3/T4/T5/T6 and the full SceneExtractor (T7) are incremental widenings behind an already-rendering pipeline. First-light is early, not gated behind the entire mesh/material/light stack.

## Open decisions for the user

1. **First-light gate: unlit-flat first (positions+camera+flat color via the T7a minimal extractor) then enable normals+lights, or lit-from-the-start?**
   - unlit-flat gate (T7a minimal extractor) then lit follow-up
   - lit from the start
   - *Recommendation:* unlit-flat gate via the T7a minimal extractor. The extraction API is identical either way; only the PoC acceptance milestone differs. Unlit-flat de-risks first-light from both normal-generation correctness AND the full mesh/material/light stack, and is now an explicit vertical-slice task (T7a) so a triangle is on screen after T0+T1+T7a+T9.

2. **Are textures in scope for the PoC first corpus render?**
   - untextured flat/lit (url plumbed at the seam but not loaded)

## Risks

- Per-path INCREMENTAL transform staleness is now structurally closed (T8 re-accumulates worldM along the stored PathKey, never reading the first-path ctx.worldTransform), but it depends on the entry-matrix cache + transformDeps reverse index being correctly populated during the walk. If a Transform is added/removed mid-scene (DirtyChildren) the cache for that prefix must be rebuilt before the next transform-only tick or a stale entry matrix is used — covered by T8's DirtyChildren subtree re-walk but needs an explicit test ordering DirtyChildren-then-transform in the same frame.
- DirtyChildren/PathKey lifetime: PROTO-instance expansion and Script/createNode dynamic node creation mid-cascade would create nodes with no prior PathKey/reverse-index entry until a DirtyChildren re-walk. Confirm the cascade in THIS codebase cannot create nodes mid-tick (it appears event-only over a pre-expanded graph); if it can, the extractor must treat unknown changed nodes as a full-walk trigger. Structural mutation (deferred M2C-2) invalidates PathKeys — documented.
- Normals remain the #1 content-completeness risk for lit corpus scenes: real IFS frequently omit Normal and rely on creaseAngle smoothing. Mitigation: FLAT face normals + two-sided lighting first; creaseAngle smoothing is a follow-up. Lit PoC is flat-shaded until then.
- Visibility-aware traversal correctness: the DFS MUST special-case Switch(whichChoice, default -1)/LOD(children/range) by nodeTypeName before the generic loop — verified the blind forEachChild would draw all Switch choices, the first child on whichChoice=-1, and all LOD levels. Designed into T7 with explicit acceptance tests; do not let an implementer reuse forEachChild for these two types.
- DirectionalLight global=false default (verified) means a naive scene-wide light over-illuminates. The PoC's correct minimum is the NavigationInfo headlight fallback (T12); full per-light scope-tagging is the documented fast-follow (reverse index makes it cheap). Surfaced-but-not-enforced: light scope, TextureTransform, MultiTexture per-stage blend, OrthoViewpoint/GeoViewpoint projection, Billboard view-facing rotation — all documented limitations / BACKLOG candidates.
- std::any_cast per field read (geombounds::getField) is slow for megavertex meshes; GeomId.contentVersion + the mesh cache (T7) must ensure unchanged meshes are NOT re-extracted every tick — and updatedGeometry is now gated on content-DirtyField NOT DirtyBounds (verified BoundsSystem.recomputeUp + classifyDirty 'scale' pollute DirtyBounds during pure transform animation), closing the every-frame-reupload trap.
- First-path semantics retained for ctx.worldTransform()/worldBounds()/PickSystem::worldOf/viewMatrix() — correct for ~97.8% non-shared and for propagate+camera (never USE-shared under two frames). Renderers consume the per-path RenderItem stream + sceneWorldBounds() instead; documented as first-path at every such accessor.
- GLFW>=3.4 native Wayland needs wayland/wayland-protocols/libxkbcommon/EGL + (on proprietary NVIDIA) libnvidia-egl-wayland dev headers; FetchContent CONFIGURES fine then FAILS at build with cryptic errors if missing. Mitigation: X11=ON kept as XWayland safety net, exact Arch prereqs in the README, glad shipped PRE-GENERATED as real committed files (not a generator FetchContent). All firewalled behind X3D_CPP_BUILD_POC=OFF so default build/golden/ctest never touch it.
- Black-screen silent-failure modes (wrong winding => backface-culled, wrong matrix order => off-screen, column-major mismatch): mitigated by GL_KHR_debug callback + forced-clear sanity step in M0, GL_CULL_FACE OFF through M1/M2, and column-major Mat4 uploaded directly via m.m.data() (verified GL-native). X3D fieldOfView is MIN-dimension FOV (verified default 0.7854) — the PoC maps it to the shorter window dimension; treating it as vertical FOV makes wide windows look wrong.
- The one-delta-per-tick seam contract is an ASSERTED invariant, not enforced by the producer (tick() auto-clears dirty_ at line 103). A future fixed-timestep consumer ticking N times per frame would drop the first N-1 ticks' changes if it batches. Documented loudly; if a real consumer needs batching, the fix is an extractor-side accumulating pending-change set folded every tick (out of PoC scope).
