# Week Plan — X3D Browser Runtime (week of 2026-06-03)

**Context:** Event cascade foundation (sub-project #1) is shipped (commit `a9328ef`).
This week advances toward a usable browser runtime. See the roadmap in
`2026-06-03-event-system-design.md`.

**Architecture decision (binding):** Keep the OOP node model + reflection as the
source of truth. Adopt ECS-*flavored* patterns — a `System` abstraction over node
collections, data-oriented batching at hot paths — but NOT a full ECS runtime and
NOT entities-replacing-nodes. Rationale: X3D is a retained-mode scene-graph spec
(node identity, DEF/USE sharing, field ROUTEs); no major X3D runtime uses ECS. A
real ECS, if ever, would be a runtime-side projection under the node model, taken
only when profiling a large dynamic scene proves OOP traversal is the bottleneck.

## Day 1 — Harden & integrate the foundation
- **Scene → EventGraph bridge** (keystone): resolve `Route.fromNode/toNode` DEF
  names to `FieldAddress` endpoints via the Scene DEF table; build/populate an
  `X3DExecutionContext` from a parsed `Scene`. Makes loaded `.x3d` documents
  tickable, not just hand-wired graphs.
- **ROUTE validation at registration:** reject unknown field, type-mismatched, or
  wrong-direction routes (no routing *to* outputOnly / *from* inputOnly) with a
  clear error — instead of a mid-cascade `any_cast` throw.
- Open the `modernize-x3d-spec → main` PR.
- *Verify:* tests that load a scene with ROUTEs and tick it; bad routes rejected.

## Day 2 — System seam + first interpolators
- Generalize `ActiveNode` into a small **`System`** abstraction: `update(now, ctx)`
  over a *collection* of attached nodes (event-driven systems register handlers;
  time-driven systems run per tick). Port `TimeSensorBehavior` /
  `PositionInterpolatorBehavior` onto it (behavior-preserving).
- Factor the shared interpolator `key`/`keyValue` lookup into a reusable helper.
- Implement `ScalarInterpolator`, `ColorInterpolator` (HSV-correct) as systems, TDD.

## Day 3 — Finish the interpolator family
- `OrientationInterpolator` (SLERP), `CoordinateInterpolator` /
  `CoordinateInterpolator2D`, `NormalInterpolator`, `PositionInterpolator2D`.
- *Verify:* per-interpolator cascade tests + one multi-interpolator scene.

## Day 4 — TimeSensor / X3DTimeDependentNode lifecycle
- Full semantics: `enabled`, `loop`, `startTime`/`stopTime` gating,
  `pauseTime`/`resumeTime`, outputs `isActive`, `isPaused`, `elapsedTime`,
  `cycleTime`, `time`. Put the lifecycle in a reusable `X3DTimeDependentNode`
  base (audio/movie inherit it later).
- *Verify:* clock-driven tests for start/stop/loop/pause edge cases.

## Day 5 — Document-to-animation demo + polish
- End-to-end: parse an X3D file (TimeSensor + interpolator + Transform + ROUTEs)
  → build context → `tick()` loop → Transform animates. Optional tiny `x3d-tick`
  example program.
- CI wiring for new tests; update design doc + memory; regenerate golden if any
  codegen changed.

## Out of scope this week
Sensors needing a renderer/picking (Touch/Proximity/Visibility/Collision),
Script (JS VM), followers, particles, physics, media decode, and the renderer
itself — each a later sub-project per the roadmap.
